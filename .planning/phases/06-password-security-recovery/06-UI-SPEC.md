## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps in hint rows |
| sm | 8px | Gap between strength bar segments |
| lg | 16px | Card inner padding horizontal, gap between form fields |
| xl | 24px | Card inner padding vertical (p-6) |
| 2xl | 32px | Card outer padding (p-8 on sm+) |
| 3xl | 48px | Vertical breathing room between card and viewport edge |

Exceptions:

| Value | Justification |
|-------|---------------|
| 2px | Visual micro-gap between strength bar segments for segment separation; not a layout token, purely decorative spacing within the PasswordStrengthMeter component |
| 12px | Matches existing `gap-3` convention used on the register page for form-field vertical gaps; retained for consistency with the existing register form, not used as a general layout token |
| 44px | Touch-target minimum height (h-11 class) for inputs and buttons; accessibility requirement, not a layout spacing token |

### PasswordStrengthMeter Specification

**Layout:** Vertical stack below password input field.

1. **Color bar:** 4 equal-width segments, 8px height, 4px border-radius per segment. Segments fill left-to-right based on score. Gap between segments: 2px (exception — see Spacing Scale).
2. **Hint text:** Below color bar. Single line: "Minimo 10 caracteres" with checkmark icon (lucide `Check`, 14px) when met, or circle icon (lucide `Circle`, 14px) when not met.

**Scoring logic (frontend, length-only):**
- Score 0: empty input
- Score 1: 1-5 characters (1 segment filled, destructive color)
- Score 2: 6-9 characters (2 segments filled, amber)
- Score 3: 10-14 characters (3 segments filled, yellow)
- Score 4: 15+ characters (4 segments filled, primary color)

**Interaction:** Updates on every keystroke (controlled input onChange). No debounce needed.

**Size:** Full width of parent form. Bar height 8px. Hint text 12px. Total component height approximately 36px.

## Page Layouts

### Forgot Password Page (`/forgot-password`)

**Focal point:** The email input field and "Enviar Link" submit button — the single action the user must complete.

**Layout:** Centered card on full-viewport background. Identical to register page structure.

```
[Background: --palette-background, min-h-screen, flex center]
  [Card: max-w-sm sm:max-w-md, rounded-xl, p-6 sm:p-8, shadow-lg]
    [Logo: logo_horizontal.png, max-w-[280px], centered, mb-6]
    [Heading: "Recuperar Senha", text-lg font-semibold, centered]
    [Subtitle: "Digite seu email para receber o link de recuperacao", text-sm, muted-fg, centered, mb-6]
    [Form]
      [Email input: h-11, full-width, palette-surface bg, palette-border]
      [Submit button: h-11, full-width, palette-primary bg, white text, "Enviar Link"]
    [/Form]
    [Back link: "Voltar ao login", text-xs, muted-fg, centered, mt-4]
  [/Card]
```

**Success state (after submit):** Replace form content with confirmation message. Card stays visible, no page navigation.

```
[Checkmark icon: lucide Mail, 48px, palette-primary, centered, mb-4]
[Heading: "Email Enviado", text-lg font-semibold, centered]
[Body: "Se o email estiver cadastrado, voce recebera um link de recuperacao.", text-sm, muted-fg, centered, mb-6]
[Back link: "Voltar ao login", text-xs, palette-primary, centered]
```

### Reset Password Page (`/reset-password?token=...`)

**Focal point:** The new password input with its strength meter — guiding the user toward a strong password choice.

**Layout:** Centered card, identical to register page structure.

```
[Background: --palette-background, min-h-screen, flex center]
  [Card: max-w-sm sm:max-w-md, rounded-xl, p-6 sm:p-8, shadow-lg]
    [Logo: logo_horizontal.png, max-w-[280px], centered, mb-6]
    [Heading: "Nova Senha", text-lg font-semibold, centered]
    [Subtitle: "Defina sua nova senha", text-sm, muted-fg, centered, mb-6]
    [Form]
      [Password input: h-11, full-width, autoComplete="new-password"]
      [PasswordStrengthMeter]
      [Confirm password input: h-11, full-width, autoComplete="new-password"]
      [Submit button: h-11, full-width, palette-primary bg, white text, "Redefinir Senha"]
      [Error display: text-xs, destructive, centered]
    [/Form]
  [/Card]
```

**Token invalid state:** Same pattern as register page invalid state. Centered card with destructive-colored error message.

**Success state:** After successful reset, redirect to login page (router.push to `/`). Show toast notification "Senha alterada com sucesso" via existing toast system.

### PasswordLoginForm Modification

Add below the password input, before the submit button:

```
[Link: "Esqueceu a senha?", text-xs, muted-fg, text-right, w-full, mt-[-4px]]
```

Link navigates to `/forgot-password`. Uses Next.js `Link` component with locale prefix.

## Interaction States

### Forgot Password Form

| State | Visual |
|-------|--------|
| Default | Email input empty, submit button enabled |
| Loading | Submit button shows "..." text, disabled, opacity-50 |
| Success | Form replaced with confirmation message (always, regardless of email existence) |
| Error (network) | Red error text below submit: "Erro ao enviar. Tente novamente." |

### Reset Password Form

| State | Visual |
|-------|--------|
| Token loading | Centered "Carregando..." text in muted-fg |
| Token invalid | Destructive-colored message in centered card |
| Default | Password fields empty, strength meter at 0 segments, submit enabled |
| Typing password | Strength meter updates in real-time per keystroke |
| Password mismatch | Error shown on submit attempt, not inline |
| Loading (submit) | Submit button shows "...", disabled, opacity-50 |
| Success | Redirect to login + toast notification |
| Server error (common password) | Red error text: "Esta senha e muito comum. Escolha outra." |

### Strength Meter

| Score | Segments Lit | Color | Hint Icon |
|-------|-------------|-------|-----------|
| 0 | 0 of 4 | all border-color | Circle (muted-fg) |
| 1 | 1 of 4 | destructive | Circle (muted-fg) |
| 2 | 2 of 4 | amber-500 | Circle (muted-fg) |
| 3 | 3 of 4 | yellow-500 | Check (primary) |
| 4 | 4 of 4 | primary | Check (primary) |

Score 3+ means the 10-character minimum is met. Hint icon changes from Circle to Check at score 3.

## Accessibility

| Requirement | Implementation |
|-------------|----------------|
| Error announcements | `role="alert"` on all error messages (matches existing pattern) |
| Strength meter ARIA | `role="meter"`, `aria-valuenow={score}`, `aria-valuemin={0}`, `aria-valuemax={4}`, `aria-label="Forca da senha"` |
| Hint text association | Strength meter linked to password input via `aria-describedby` |
| Focus management | On token-invalid state, focus the error message container |
| Form labels | Inputs use `placeholder` as visible label (matches existing pattern). Add `aria-label` matching placeholder text for screen readers |
| Color not sole indicator | Strength hint text ("Minimo 10 caracteres") + checkmark icon supplement color bar |

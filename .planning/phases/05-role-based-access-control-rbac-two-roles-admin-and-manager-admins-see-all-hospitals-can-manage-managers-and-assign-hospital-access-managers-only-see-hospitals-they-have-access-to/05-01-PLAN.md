    Create `api/migrations/005_create_users.up.sql` per D-05/D-12 with:
    ```sql
    CREATE TABLE IF NOT EXISTS users (
        id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
        email       TEXT NOT NULL UNIQUE,
        name        TEXT,
        picture     TEXT,
        role        TEXT NOT NULL DEFAULT 'manager' CHECK (role IN ('admin', 'manager')),
        created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
        updated_at  TIMESTAMPTZ NOT NULL DEFAULT NOW()
    );

    CREATE TABLE IF NOT EXISTS user_hospitals (
        user_id     UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
        hospital_id UUID NOT NULL REFERENCES hospitals(id) ON DELETE CASCADE,
        created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
        PRIMARY KEY (user_id, hospital_id)
    );

    CREATE INDEX IF NOT EXISTS idx_user_hospitals_user ON user_hospitals(user_id);
    CREATE INDEX IF NOT EXISTS idx_user_hospitals_hospital ON user_hospitals(hospital_id);
    ```

    Create `api/migrations/005_create_users.down.sql`:
    ```sql
    DROP TABLE IF EXISTS user_hospitals;
    DROP TABLE IF EXISTS users;
    ```

    **1. Create `api/internal/repository/user.go`:**

    Follow the same pattern as HospitalRepo. Struct with `pool *pgxpool.Pool`, constructor `NewUserRepo(pool)`.

    Methods:
    - `FindByEmail(ctx, email) (*model.User, error)` -- SELECT from users WHERE email = $1. Return nil, nil if not found (not an error).
    - `Upsert(ctx, user *model.User) (*model.User, error)` -- `INSERT INTO users (email, name, picture, role) VALUES ($1,$2,$3,$4) ON CONFLICT (email) DO UPDATE SET name=COALESCE(EXCLUDED.name, users.name), picture=COALESCE(EXCLUDED.picture, users.picture), role=EXCLUDED.role, updated_at=NOW() RETURNING id, email, name, picture, role, created_at, updated_at`. Uses COALESCE so admin re-login doesn't blank out name/picture if they already exist.
    - `CreateManager(ctx, email string) (*model.User, error)` -- INSERT with role='manager', email only. Return conflict error if exists.
    - `ListAll(ctx) ([]model.UserWithHospitals, error)` -- SELECT all users LEFT JOIN user_hospitals LEFT JOIN hospitals. Group hospitals per user in Go code (iterate rows, build map by user ID).
    - `UpdateHospitalAccess(ctx, userID string, hospitalIDs []string) error` -- In a transaction: DELETE FROM user_hospitals WHERE user_id=$1, then INSERT each hospital_id. Use `pool.Begin(ctx)` for transaction.
    - `Delete(ctx, userID string) error` -- DELETE FROM users WHERE id=$1 (CASCADE handles user_hospitals).

<objective>
Fix Google OAuth client_id missing at Docker build time.

Purpose: Next.js inlines NEXT_PUBLIC_* env vars at build time via string replacement. The current setup only sets NEXT_PUBLIC_GOOGLE_CLIENT_ID at runtime in docker-compose.yml, but by then the value is already baked into the JS bundle as an empty string. The fix passes the value as a Docker build arg so it is available during `npm run build`.

Output: Updated Dockerfile and docker-compose.yml that correctly propagate the Google client ID into the Next.js build.
</objective>

  <files>frontend/Dockerfile, docker-compose.yml</files>
  <action>
1. Edit `frontend/Dockerfile` — In the builder stage (before `RUN npm run build`), add:
   ```
   ARG NEXT_PUBLIC_GOOGLE_CLIENT_ID
   ENV NEXT_PUBLIC_GOOGLE_CLIENT_ID=$NEXT_PUBLIC_GOOGLE_CLIENT_ID
   ```
   Place these two lines after `COPY . .` and before `RUN npm run build`. This ensures the env var is available when Next.js performs its static string replacement during build.

2. Edit `docker-compose.yml` — Under the frontend service's `build:` section, add an `args:` block to forward the host env var:
   ```yaml
   build:
     context: ./frontend
     dockerfile: Dockerfile
     args:
       NEXT_PUBLIC_GOOGLE_CLIENT_ID: ${GOOGLE_CLIENT_ID:-}
   ```
   Keep the existing `environment` section as-is (runtime env vars are still useful for server-side code in standalone mode).
  </action>
